## Open findings

### P1 — `cli-response-envelope.json` command enum is missing MCP tool names

**File:** `docs/schemas/cli-response-envelope.json:14-18`

Current enum:
```json
["extract", "extract-batch", "preview", "transcribe-run", "open-last", "artifacts"]
```

`callMCPTool` emits these envelope `command` values that are not in the enum:

| MCP command string | Emitted at |
|---|---|
| `doctor` | `cmd/frames/main.go:4082` |
| `get_latest_artifacts` | `cmd/frames/main.go:4406,4408` |
| `get_run_artifacts` | `cmd/frames/main.go:4437,...` |
| `transcribe_run` | `cmd/frames/main.go:4485,4487` |
| `prefs_get` | `cmd/frames/main.go:4495` |
| `prefs_set` | `cmd/frames/main.go:4522,4530` |

Note `transcribe_run` (underscore, MCP) vs `transcribe-run` (hyphen, CLI) — two distinct strings, both real.

**Why it matters:** `docs/AGENT_INTEGRATION.md:76-89` presents this envelope as the MCP tool payload shape. A strict JSON-schema validator would reject real MCP responses.

**Fix options:**
1. **Extend the enum** to include all MCP tool names above. Smallest delta, but couples the schema to the MCP tool list — adding a new tool means updating the schema too.
2. **Drop the enum** (`command` becomes unconstrained string), add `$comment` pointing at `cmd/frames/main.go` as source of truth. Most forgiving, matches the existing `$comment` pattern for `doctor --json`.
3. **Rename MCP commands to hyphenated CLI form** in `callMCPTool`. Breaking change for any MCP client that pattern-matches on `command`. Don't do this without a version bump.

Recommendation: option 2 (drop the enum, add a comment). The schema's job is to document shape, not enumerate every tool name.

**Reproduce:**
```bash
# Show the enum
sed -n '14,18p' docs/schemas/cli-response-envelope.json
# Show MCP emissions
grep -n 'buildAutomationEnvelope(' cmd/frames/main.go | grep -v '"extract"\|"extract-batch"\|"preview"\|"transcribe-run"\|"open-last"\|"artifacts"'
```

### P2 — CHANGELOG claims a `version` subcommand that doesn't exist

**File:** `CHANGELOG.md:61` (v0.2.3 section)

Text reads:
> `--version` flag and version subcommand with goreleaser-populated ...

Only `--version` is wired (`cmd/frames/main.go:334` sets `root.Version`, which makes cobra auto-add the flag only). There is no `root.AddCommand(versionCommand())` anywhere in `main.go:338-357`.

**Reproduce:**
```bash
go build -o /tmp/framescli ./cmd/frames
/tmp/framescli --version   # prints "framescli dev" — works
/tmp/framescli version     # Error: unknown command "version" — drift
```

**Fix options:**
1. **Strike "and version subcommand"** from the 0.2.3 entry. One-line CHANGELOG edit.
2. **Add a `version` subcommand** that prints the same string. Trivial (roughly 8 lines) but creates two ways to ask for the same thing. Only do this if you actually want both surfaces.

Recommendation: option 1. The CHANGELOG is historical record — fixing the claim is a doc correction, not a feature revert.
